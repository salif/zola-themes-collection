Intelligent Sidenotes System

The Pechersk theme features a sophisticated sidenotes system that adapts to different screen sizes and provides intelligent positioning algorithms.

Auto-Positioning Algorithm

The system automatically alternates sidenotes between left and right sides1. This creates visual balance and prevents overcrowding2 on either side of the content3.

The algorithm considers previous placements4 and maintains alternating patterns5 for optimal reading experience6.

How Auto-Positioning Works

When you create a footnote without explicit positioning7, the system:

  1. Checks for explicit positioning (left/right keywords)
  2. Calculates optimal side based on alternating logic
  3. Updates internal state for next footnote
  4. Applies responsive behavior based on screen size

Explicit Positioning Control

Sometimes you need precise control over sidenote placement8. Use explicit positioning when you want specific visual arrangements9.

Left-Side Positioning

You can force notes to the left side using left in the footnote ID10. This is useful for complementary information11 that relates to specific content on the right12.

Right-Side Positioning

Similarly, force right-side placement using right in the footnote ID13. Right-side notes work well for additional details14 or elaborative content15.

Mixed Positioning Strategy

Combine auto and explicit positioning strategically16. Use explicit positioning for important notes17 while letting the system handle routine footnotes18. This creates intentional visual hierarchy19.

Responsive Behavior Showcase

Desktop Experience (1024px+)

On desktop screens, sidenotes float beside the main content in dedicated grid areas. The intelligent positioning creates a magazine-like reading experience27.

Tablet Experience (768px-1023px)

On tablet devices, all sidenotes consolidate to the right side for optimal touch interaction28. This prevents interference with touch scrolling29.

Mobile Experience (767px-)

On mobile devices, sidenotes become inline footnotes at the bottom of the content30. This ensures accessibility31 and prevents layout issues32.

Performance Characteristics

The sidenotes system is designed for optimal performance33:

  • Lightweight JavaScript: ~2KB compressed
  • Efficient algorithms: O(1) positioning decisions
  • Minimal DOM manipulation: Only updates when necessary
  • Debounced resize handling: Smooth window resizing34
  • Progressive enhancement: Works without JavaScript35

Accessibility Features

Full accessibility support includes36:

  • Screen reader compatibility: Proper ARIA labels
  • Keyboard navigation: Tab and focus management
  • High contrast support: Theme-aware colors
  • Touch-friendly targets: Optimized for mobile interaction
